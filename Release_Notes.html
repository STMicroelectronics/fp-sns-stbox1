Release Notes for FP-SNS-STBOX1 Software for STM32

Copyright © 2024 STMicroelectronics

Purpose

The FP-SNS-STBOX1 is STM32Cube function pack for the Pro Mode of wireless multi sensor development kits The purpose of this functional pack is to provide simple applications and examples that show how to build custom applications for STEVAL-MKBOXPRO and STEVAL-STWINBX1 boards.

The expansion is built on STM32Cube software technology to ease portability across different STM32 microcontrollers.

FP-SNS-STBOX1 software features:

  • Complete examples and applications to develop node with BLE connectivity, analog microphone, environmental and motion sensors, and perform real-time monitoring of sensors and audio data

  • Example of how create one implementation based on transmitting the data with BLE

  • Example of how create one application for allowing Firmware Over the Air update

  • Compatible with STBLESensor application for Android/iOS, to perform sensor data reading, audio and motion algorithm feature demo, and firmware update over the air (FOTA)

  • Easy portability across different MCU families, thanks to STM32Cube

  • Free, user-friendly license terms

This firmware package includes Components Device Drivers, Board Support Package and example application for the STMicroelectronics:

  • STEVAL-MKBOXPRO (SensorTile.box-Pro) evaluation board that contains the following components:

    • MEMS sensor devices: STTS22H, LPS22DF, LSM6DSV16X, LIS2DU12, LIS2MDL

    • Dynamic NFC tag: ST25DV04K

    • Digital Microphone: MP23db01HP

    • BlueNRG-LP Bluetooth Low Energy System On Chip

  • STEVAL-STWINBX1 (STWIN.box) evaluation board that contains the following components:

    • MEMS sensor devices: IIS2DLPC, IIS2MDC, IIS3DWB, ISM330DHCX, IIS2ICLX, ILPS22QS, STTS22H

    • dynamic NFC tag: ST25DV64

    • analog (IMP23ABSU) and digital (IMP34DT05) microphones

    • BlueNRG-2 Bluetooth Low Energy System On Chip

This package contains the following projects:

  • BLEDualProgram: Secure (PIN) BLE Firmware Over The Air update with Dual Boot option for Firmware Rollback (for STEVAL-MKBOXPRO only)

  • BLEMLC: Example on how to program the ISM330DHCX (for STEVAL-STWINBX1) or LSM6DSV16X (for STEVAL-MKBOXPRO) Machine Learning Core (MLC)

  • BLEPiano: The application explains how using bluetooth it is possible to play Music Notes (for STEVAL-MKBOXPRO only)

  • BLESensorsPnPL: Simplest way for BLE trasmission of Sensor’s data with PnPL-Like

  • BLEDefaultFw: Default firmware for STEVAL-STWINBX1 (for STEVAL-STWINBX1 only)

  • NFC_FTM: Example on how to use NDEF and FTM

  • SDDataLogFileX: SD-Card data Log using filex

  • ExampleCubeMxDataLog: STM32CubeMX example code (for STEVAL-STWINBX1 only)

Known Issues

Known Limitations

With some Android phones there are some compatibility issues for:

  • using the PIN for BLE security connection (BLEFOTA/BLEDualProgram application). In this case disable the STBOX1_BLE_SECURE_CONNECTION on respective Inc/STBOX1_config.h file
  • for forcing a full BLE rescan (BLEMLC/BLELowPowerRToS/BLESensors applications). In this case disable the BLE_FORCE_RESCAN on their Inc/STBOX1_config.h files

In all these situations, before to connect to the SensorTile.box, each time you change the running application, clean the Device Cache for forcing a rescan of BLE services:

This method will be no more longer available for Android Q

Known Limitations

With some Android phones there are some compatibility issues for:

  • using the PIN for BLE security connection (BLEFOTA/BLEDualProgram application). In this case disable the STBOX1_BLE_SECURE_CONNECTION on respective Inc/STBOX1_config.h file
  • for forcing a full BLE rescan (BLEMLC/BLELowPowerRToS/BLESensors applications). In this case disable the BLE_FORCE_RESCAN on their Inc/STBOX1_config.h files

In all these situations, before to connect to the SensorTile.box, each time you change the running application, clean the Device Cache for forcing a rescan of BLE services:

This method will be no more longer available for Android Q

Backward Compatibility

None

Dependencies

This software release is compatible with:

  • ST BLE Sensor Android application V4.6.0
  • ST BLE Sensor iOS application V4.6.0
  • ST Unicleo-GUI PC application V1.11.0

Main Changes

Maintenance release

  • Updated drivers (components and SensorTile.box)
  • Changed the name of the DataLog application to SDDataLog
  • Removed Vibration monitor program for Machine Learning Core (BLEMLC application)
  • Introduced the capability to program the Finite State Machine (BLEMLC application)
  • Introduced the new Bluetooth characteristics for displaying the output of MLC and FSM (BLEMLC application)
  • Improved the USBD recognition
